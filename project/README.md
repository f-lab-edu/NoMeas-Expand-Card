## **프로젝트 목적**

**디자인 시스템을 기반으로 UX 규칙을 시스템화하고,**

**LLM을 ‘실험 속도를 높이는 가속 레이어’로 활용하여 UX 품질을 유지한**

**논코드 기반의 빠른 프로토타입 실험 환경을 구축**

```
💡 LLM 자체가 목적이 아니라, UX Engineering 관점의 실험 인프라 설계
```

## **기대효과**

### **1. 개발 리소스에 종속되지 않는 UX 실험 환경**

- 비개발자 직군도 논코드로 직접 프로토타이핑 가능
- 개발자 가용 시간에 영향받지 않는 빠른 UX 가설 검증

### **2. 프로토타입의 실서비스 자산화**

- 디자인 시스템 컴포넌트 기반으로 프로토타입 생성
- 실험 결과를 실제 서비스 컴포넌트로 즉시 전환 가능

### **3. UX 품질이 보장되는 빠른 실험**

- 실험 속도와 UX 일관성 동시 확보
- 누구나 만들어도 동일한 UX 품질 유지

## **기술 스택 (요약)**

- Bun

  프로토타이핑/실험 프로젝트 범위에서 통합 툴체인의 효용을 검증하고,  
  결과를 설계 판단 근거로 남기기 위함

- 렌더링: React

  컴포넌트 기반 UI, JSON → UI 안정적 매핑

- 스타일링: Tailwind

  정의되지 않은 컴포넌트 요청에도 디자인 시스템 foundation을 벗어나지 않도록,  
  유틸리티 클래스 표준화를 통해 조합 기반 UI 생성을 가능하게 하기 위함

- 문서화: Storybook

  디자인 시스템 공통 문서화 및 기준점

  디자이너·개발자·LLM이 동일한 컴포넌트 정의 공유

- Server-side LLM API (필요성 검증 대상)

  자연어 UX 의도 → UI 구조 생성

  UX 규칙 위반 시 재생성/보정

## **구축 방법**

### **옵션 A. 직접 구축**

- 디자인 시스템 문서·카탈로그 직접 구현
- 자유도 높음
- 초기 비용 및 기간 증가

### **옵션 B. Storybook + [Story ui](https://southleft.com/insights/design-systems/introducing-story-ui-accelerating-layout-generation-with-ai-mcp/) 기반 구축**

- Storybook을 디자인 시스템 문서화 인프라로 활용
- UX Constraint Layer & LLM 연동에 집중
- 프로젝트 타임라인 단축
